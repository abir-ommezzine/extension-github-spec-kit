# Backend Architecture - Agentic Pipeline

This directory contains the backend implementation of the GitHub Spec Kit, an agentic pipeline designed to transform raw technical documentation into structured, validated architectural specifications.

## 🤖 Agent Overview

The pipeline consists of three primary specialized agents, each with its own service, toolset, and evaluation framework.

### 1. Parsing Agent
**Role**: Expert Technical Ingestion Agent. Its goal is to transform raw markdown documents into a strict, unified JSON compliance graph.

- **Architecture**: Uses a hybrid approach combining deterministic Python AST analysis and LLM-based extraction.
- **Tools**:
    - `pre_parse_markdown_to_sections`: Decomposes raw markdown into physical sections.
    - `calculate_file_hash`: Ensures document versioning and integrity.
- **Metrics (defined in `metrics.py` & `evaluation_service.py`)**:
    - **SAR (Schema Adherence Rate)**: Validates JSON structure against Pydantic schemas.
    - **SIR (Structural Integrity Recall)**: Measures the percentage of required physical sections preserved.
    - **TFS (Text Fidelity Score)**: Ensures raw content is not altered or summarized.
    - **ExR (Extraction Recall)**: Measures the ability to capture all questions from the source.
    - **GRI (Graph Relational Integrity)**: Validates that all graph edges point to existing nodes.
- **Resources**:
    - `backend/app/resources/sdd_templates.json`: Defines required sections and expected element types per document type.
- **System Prompt**: Generated by `get_parsing_agent_prompt` in `app/core/prompts.py`.

---

### 2. Summary Agent
**Role**: Senior Architecture Synthesis Engineer. Analyzes the parsed semantic graph to generate a high-level strategic technical note.

- **Architecture**: Consumes the pruned output of the Parsing Agent and integrates project maturity KPIs.
- **Tools**:
    - `SummaryPrunerService.prune_payload`: Performs semantic pruning of the JSON payload to optimize LLM context.
    - `ParsingEvaluatorService`: Extracts management KPIs to ground the summary in factual project maturity.
- **Metrics (defined in `metrics.py` & `evaluation_service.py`)**:
    - **WCA (Word Count Adherence)**: Verifies the executive brief is within the 30-150 word limit.
    - **GSC (Graph Stack Content)**: Measures the alignment between the summarized stack and the parsed elements.
    - **MAC (Maturity Assessment Coherence)**: Evaluates if the narrative correctly accounts for structural gaps.
- **Resources**:
    - `backend/app/resources/summary_spec.json`: Defines the target output specification for the synthesis.
- **System Prompt**: Generated by `get_summary_agent_prompt` in `app/core/prompts.py`.

---

### 3. Glossary Agent
**Role**: Semantic Normalization & Technology Anchor Agent. Maps and defines business and technical terms to eliminate semantic hallucinations.

- **Architecture**: Implements strict geometric anchoring, linking every definition to a specific node identifier from the Parsing Agent's graph.
- **Tools**:
    - `GlossaryHarvesterService.harvest_candidates`: Deterministically extracts candidate terms, acronyms, and implicit norms.
- **Metrics (defined in `metrics.py` & `evaluation_service.py`)**:
    - **ATA (Anti-Tautology Adherence)**: Ensures definitions do not repeat the term itself.
    - **CAP (Contextual Anchor Precision)**: Validates that `contextual_anchor` values match real nodes/sections from the parsed graph.
- **Resources**:
    - `backend/app/resources/glossary_spec.json`: Governance rules for term classification.
    - `backend/ressources/topological_anchors_cache.json`: Closed cache of valid geometric anchors.
- **System Prompt**: Generated by `get_glossary_agent_prompt` in `app/core/prompts.py`.

---

## 🛠️ Core Infrastructure

### Evaluation & Metrics
- **`backend/app/core/metrics.py`**: Contains the pure mathematical implementations of all reliability and precision metrics.
- **`backend/app/services/evaluation_service.py`**: Orchestrates the evaluation process, applying metrics to agent outputs to determine the `readiness_status` (e.g., `READY_FOR_EXECUTION`, `NEEDS_REFINEMENT`, `BLOCKED`).

### Prompt Management
- **`backend/app/core/prompts.py`**: Centralized factory for system prompts. It injects dynamic specifications, candidate terms, and resource templates into the prompts to ensure deterministic LLM behavior.

### Data Schemas
- **`backend/app/schemas/`**: Contains Pydantic models (`ParsingAgentOutput`, `SummaryOutputModel`, `GlossaryOutputModel`) used for strict validation of LLM outputs.
