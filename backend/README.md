# Backend Architecture - Agentic Pipeline

This directory contains the backend implementation of the GitHub Spec Kit, an agentic pipeline designed to transform raw technical documentation into structured, validated architectural specifications.

## 🔄 Pipeline Workflow

The system implements a directed acyclic graph (DAG) using LangGraph, ensuring a deterministic flow of information from raw ingestion to final synthesis.

**Process Flow:**
`[START]` $\rightarrow$ `[Parsing Agent]` $\rightarrow$ `{ [Summary Agent], [Glossary Agent], [Diagram Agent] }` (Parallel) $\rightarrow$ `[END]`

1. **Ingestion Phase**: The **Parsing Agent** acts as the gateway, transforming raw markdown into a strict, unified JSON compliance graph.
2. **Analysis Phase (Parallel)**: Once the semantic graph is established, three specialized agents operate concurrently to extract different dimensions of the project:
    - **Summary Agent**: Strategic synthesis and high-level overview.
    - **Glossary Agent**: Semantic normalization and terminology anchoring.
    - **Diagram Agent**: Topological visualization.
3. **Convergence**: The pipeline converges these outputs into a final, validated specification package.

---

## 🤖 Agent Overview

### 1. Parsing Agent
**Role**: Expert Technical Ingestion Agent. Its goal is to transform raw markdown documents into a strict, unified JSON compliance graph.

- **Architecture**: Uses a hybrid approach combining deterministic Python AST analysis and LLM-based extraction.
- **Tools**:
    - `pre_parse_markdown_to_sections`: Decomposes raw markdown into physical sections.
    - `calculate_file_hash`: Ensures document versioning and integrity.
- **Metrics**:
    - **SAR (Schema Adherence Rate)**: Validates JSON structure against Pydantic schemas.
    - **SIR (Structural Integrity Recall)**: Measures the percentage of required physical sections preserved.
    - **TFS (Text Fidelity Score)**: Ensures raw content is not altered or summarized.
    - **ExR (Extraction Recall)**: Measures the ability to capture all questions from the source.
    - **GRI (Graph Relational Integrity)**: Validates that all graph edges point to existing nodes.
- **Resources**:
    - `backend/app/resources/sdd_templates.json`: Defines required sections and expected element types per document type.
- **System Prompt**: Generated by `get_parsing_agent_prompt` in `app/core/prompts.py`.

---

### 2. Summary Agent
**Role**: Senior Architecture Synthesis Engineer. Analyzes the parsed semantic graph to generate a high-level strategic technical note.

- **Architecture**: Consumes the pruned output of the Parsing Agent and integrates project maturity KPIs.
- **Tools**:
    - `SummaryPrunerService.prune_payload`: Performs semantic pruning of the JSON payload to optimize LLM context.
    - `ParsingEvaluatorService`: Extracts management KPIs to ground the summary in factual project maturity.
- **Metrics**:
    - **WCA (Word Count Adherence)**: Verifies the executive brief is within the 30-150 word limit.
    - **GSC (Graph Stack Content)**: Measures the alignment between the summarized stack and the parsed elements.
    - **MAC (Maturity Assessment Coherence)**: Evaluates if the narrative correctly accounts for structural gaps.
- **Resources**:
    - `backend/app/resources/summary_spec.json`: Defines the target output specification for the synthesis.
- **System Prompt**: Generated by `get_summary_agent_prompt` in `app/core/prompts.py`.

---

### 3. Glossary Agent
**Role**: Semantic Normalization & Technology Anchor Agent. Maps and defines business and technical terms to eliminate semantic hallucinations.

- **Architecture**: Implements strict geometric anchoring, linking every definition to a specific node identifier from the Parsing Agent's graph.
- **Tools**:
    - `GlossaryHarvesterService.harvest_candidates`: Deterministically extracts candidate terms, acronyms, and implicit norms.
- **Metrics**:
    - **ATA (Anti-Tautology Adherence)**: Ensures definitions do not repeat the term itself.
    - **CAP (Contextual Anchor Precision)**: Validates that `contextual_anchor` values match real nodes/sections from the parsed graph.
- **Resources**:
    - `backend/app/resources/glossary_spec.json`: Governance rules for term classification.
    - `backend/ressources/topological_anchors_cache.json`: Closed cache of valid geometric anchors.
- **System Prompt**: Generated by `get_glossary_agent_prompt` in `app/core/prompts.py`.

---

### 4. Diagram Agent
**Role**: Architecture Visualization Agent. Transforms the parsed semantic graph into valid Mermaid.js architectural diagrams.

- **Architecture**: Employs a specialized LLM pipeline with a post-processing "Cleaning" phase that applies deterministic Regexes to eliminate syntax hallucinations and ensure rendering compatibility.
- **Tools**:
    - `DiagramAgentService.clean_mermaid_code`: Sanitizes LLM-generated Mermaid code to fix common syntax errors.
- **Resources**:
    - `backend/app/resources/diagram_spec.json`: Defines the target types and specifications of diagrams to be generated.
- **System Prompt**: Generated by `get_diagram_agent_prompt` in `app/core/prompts.py`.

## 🛠️ Full Backend Architecture

### 📂 Application Core (`/backend/app`)
- `main.py`: FastAPI entry point, CORS configuration, and API routing.
- `config.py`: Global settings and environment variable management.
- `database.py`: Database connection and session lifecycle.
- `models.py`: SQLAlchemy models defining the persistence layer.
- `health_check.py`: System diagnostic endpoint for availability monitoring.
- `verify_db.py`: Utility for database connectivity and schema validation.

### 🧠 Core Logic (`/backend/app/core`)
- `llm_client.py`: Unified interface for LLM communication (Ollama/OpenAI).
- `llm_utils.py`: Shared helpers for JSON validation and LLM response parsing.
- `metrics.py`: Pure mathematical implementations of reliability and precision metrics.
- `prompts.py`: Dynamic system prompt factory for all pipeline agents.
- `test_llm.py`: Integration tests for LLM connectivity and prompt tuning.

### 🕸️ Graph Orchestration (`/backend/app/graph`)
- `state.py`: Defines the `GraphState` object shared between pipeline nodes.
- `nodes.py`: Wraps specialized services into LangGraph-compatible node functions.
- `workflow.py`: Defines the DAG topology and the parallel trifurcation logic.

### 📄 Resources & Specs (`/backend/app/resources`)
- `*.json`: Target specifications that govern agent behavior and output constraints.

### 📐 Validation Schemas (`/backend/app/schemas`)
- `*.py`: Pydantic models used for strict type-validation of LLM-generated content.

### ⚙️ Business Services (`/backend/app/services`)
- `parser_service.py`: Logic for technical document ingestion and graph construction.
- `summary_service.py`: Synthesis logic for high-level executive summaries.
- `glossary_service.py`: Normalization logic for technical and business terminology.
- `diagram_service.py`: Topological analysis and Mermaid.js code generation.
- `evaluation_service.py`: Orchestrates the application of metrics to determine agent output readiness.

### 🧰 Utilities (`/backend/app/utils`)
- `markdown_parser.py`: Low-level tools for decomposing and analyzing raw markdown.
- `summary_pruner.py`: Context optimization logic for reducing LLM token usage.
- `glossary_tools.py`: Deterministic tools for extracting candidate terms.
- `diagram_tools.py`: Specialized helpers for architectural visualization.
