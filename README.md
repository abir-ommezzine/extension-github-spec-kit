# Spec Kit Project

This project implements an agentic pipeline designed to transform raw technical documentation into structured, validated architectural specifications.

## 📂 Project Structure

### ⚙️ Backend (`/backend`)
Contains the complete backend implementation of the agentic pipeline. It includes the orchestration logic (LangGraph), specialized agents (Parsing, Summary, Glossary, Diagram), and the evaluation framework. For a detailed architectural breakdown, please refer to [`backend/README.md`](backend/README.md).

### 🛠️ Configuration & Environment
- **`.env`**: Stores environment variables such as API keys, LLM configurations, and system paths. **(Do not commit this file to version control)**.
- **`.specify/`**: Along with other configuration files in the root, this directory is automatically generated by the `specify init` command from the **Spec Kit framework**. These files manage the framework's internal settings and generation rules.

### 📦 Dependencies
- **`requirements.txt`**: The primary file containing all Python dependencies required to run the backend.
- *Note: `req.txt` is a secondary file and should be ignored in favor of `requirements.txt`.*

	### 📊 Outputs Organization (`/outputs`)
	The `outputs/` directory is the centralized hub for all pipeline results, structured to separate final deliverables from intermediate artifacts.

	```text
	outputs/
	├── 📄 documents/         # FINAL deliverables (e.g., CourseHub.pdf)
	├── 💾 data/              # Work artifacts, raw data, and caches
	│   ├── 📝 markdown/      # Drafts from Doc Writer (e.g., coursehub_doc.md)
	│   ├── 🖼️ diagrams/      # Exported diagrams from Diagram Agent (PDF/PNG)
	│   └── ⚙️ topological_anchors_cache.json # Glossary agent cache
	└── 📊 evaluations/       # RAGAS benchmarks, metrics, and reports (JSON)
	```

	### 🛠️ Input & Validation Workflow
	We distinguish between official specifications and manual test cases to prevent accidental trigger of the automation pipeline.

	- **`specs/`** 🟢 **Official Source**: This directory contains the source specifications. It is monitored in real-time by the **Hook Watcher**, which automatically triggers the agent chain upon any creation or modification of a `.md` file.
	- **`test_files/`** 🧪 **Manual Testing**: Reserved exclusively for manual tests. Use this directory to store `.md` files for debugging and testing the workflow via command line **without** triggering the Hook Watcher.

## 🚀 Quick Start
1. Configure your environment variables in `.env`.
2. Install dependencies: `pip install -r requirements.txt`.
3. To run the pipeline:
   - Navigate to the backend folder: `cd backend`
   - Start the FastAPI server: `uvicorn app.main:app --reload --port 8000`
